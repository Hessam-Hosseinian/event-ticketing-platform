# Agile lifecycle summary

The project uses Scrum with a prioritized outcome backlog, three two-week sprints, explicit Ready/Done,
WIP-limited flow, independent review, stakeholder demonstrations and measurable retrospective experiments.
Epics map to bounded contexts and operational outcomes; stories map to PDF requirements and executable
acceptance examples. Documentation and failure paths are part of Done, not end-of-project cleanup.

The Jira CSV is importable evidence of Epic → Story/Task hierarchy, ownership, sprint and estimate. The
burndown CSV records daily remaining work; velocity is a forecasting input, never a performance target.
Items may move to Done only with reproducible evidence. Carry-over is re-estimated and explained in the
review/retrospective instead of rewriting the original commitment.
